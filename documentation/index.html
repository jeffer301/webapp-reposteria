Introduction & Features

La Flor de Azúcar is a full-stack web application for artisan bakeries to sell their products online. Customers browse the catalog, add items to a persistent cart, and complete checkout — receiving a printable ticket with a unique QR code by email. The store owner manages orders, products, and users through a built-in admin panel, and verifies pickups by scanning QR codes with the device camera.

Built with Angular 21 + Node.js / Express + PostgreSQL, fully Dockerized, and integrated with Wompi (Colombian payment gateway).

Key Features

  • Product catalog with images, prices & discounts
  • Category filters & keyword search
  • Persistent cart (no login required)
  • Full checkout: pickup or home delivery
  • Unique QR code per order
  • Printable order ticket modal
  • Transactional email notifications
  • Public order status verification page
  • Admin panel: orders, products, categories, users
  • Live QR scanner via device camera
  • Wompi payment gateway integration
  • PWA — installable on mobile & desktop

Installation

Method A — Manual Installation

Backend

1. Copy the environment template:
cd backend
cp .env.example .env
# On Windows:
copy backend\.env.example backend\.env
2. Open backend/.env and fill in your PostgreSQL credentials. See Environment Variables for the full reference.
3. Install dependencies and run database migrations:
npm install
npm run migrate

This creates all tables, indexes, and inserts demo data: 5 categories, 8 products, and 1 admin user (admin@bakery.com / Admin123).

4. Start the backend server:
npm run dev
# API running at http://localhost:3000
# Health check: http://localhost:3000/health

Frontend

5. Open a second terminal in bakery-frontend/:
cd bakery-frontend
npm install
ng serve
# App available at http://localhost:4200
bakery-frontend/proxy.conf.json automatically forwards all /api/* requests to http://localhost:3000 — no CORS configuration needed in development.

Method B — Docker (Recommended)

1. Copy the environment template:
cp backend/.env.example backend/.env
# On Windows:
copy backend\.env.example backend\.env
2. Edit backend/.env — at minimum set DB_PASSWORD and JWT_SECRET.
3. Build and start all services:
docker compose up --build

This single command:

  • Builds backend (Node.js) and frontend (Angular → Nginx) images
  • Downloads the official PostgreSQL 16 Alpine image
  • Runs database migrations automatically on first start
  • Seeds demo data (categories, products, admin user)
  • Starts all three containers with health checks

Switching Payment Gateways

The payment system uses a provider pattern — switching gateways requires changing a single environment variable. The active provider is selected by PAYMENT_PROVIDER in backend/.env.

ValueGatewayStatusTarget Market
wompiWompi Active Colombia
stripeStripe Stub — ready to extend Global
mercadopagoMercadoPago Stub — ready to extend Latin America

Activating Wompi (Default)

Get your sandbox keys at comercios.wompi.co.

# backend/.env
PAYMENT_PROVIDER=wompi
WOMPI_API_URL=https://sandbox.wompi.co/v1
WOMPI_PUBLIC_KEY=pub_test_xxxxxxxxxxxxxxxx
WOMPI_PRIVATE_KEY=prv_test_xxxxxxxxxxxxxxxx

For production, change WOMPI_API_URL to https://production.wompi.co/v1 and replace the test keys with your live keys.

Extending to Stripe or MercadoPago

Stub providers are located at backend/src/services/providers/. Each extends PaymentProvider.js and must implement processPayment(order): 

// backend/src/services/providers/StripeProvider.js
class StripeProvider extends PaymentProvider {
  async processPayment(order) {
    // Implement Stripe Charges API call here
    // Must return: { success: true, reference: chargeId }
  }
}

Then set PAYMENT_PROVIDER=stripe in backend/.env and restart the server.

QR Scanner

Click the QR Scanner button in the admin panel to activate the device camera. Hold a customer's order ticket QR code in front of the camera — order details appear instantly for in-store pickup verification.

The QR scanner requires camera permission in the browser. On mobile, the site must be served over HTTPS (HTTP is only allowed on localhost).

Granting Admin Access to Another User

  1. Log in as an existing admin.
  2. Go to the Users tab in the admin panel.
  3. Find the user and click Change Role → Admin.

Order Status Flow

pending → confirmed → preparing → ready → delivered
                                         ↘ cancelled  (from any state)

Each transition triggers an email notification to the customer with the new status. Customers can track their order in real time at /verify/<order-code> without needing to log in.

Option A — Self-signed certificate (internal / testing):

mkdir certs
docker run --rm -v "$(pwd)/certs:/certs" alpine sh -c \
  'apk add openssl && openssl req -x509 -nodes -days 365 \
   -newkey rsa:2048 -keyout /certs/privkey.pem \
   -out /certs/fullchain.pem -subj "/CN=YOUR_SERVER_IP"'

Option B — Let's Encrypt (production with a domain): use Certbot to generate certificates and mount them into the Nginx container at /etc/nginx/certs/.

Images uploaded through the admin panel are stored in backend/uploads/. In Docker, this folder is mounted as a named volume (uploads_data) that persists across container restarts and rebuilds.

For production at scale, consider moving images to cloud object storage (AWS S3, Cloudflare R2) by updating backend/src/controllers/uploadController.js.
